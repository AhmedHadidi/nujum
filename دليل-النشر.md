# دليل النشر — منصة نُجوم على السحابة ☁️

هذا الدليل يأخذك خطوة بخطوة لرفع الموقع على **GitHub** ونشره على **Vercel** مع قاعدة بيانات **Supabase** حقيقية مشتركة بين كل الأجهزة.

الوقت المتوقّع: **١٥–٢٠ دقيقة**. كل الخدمات المستخدمة **مجانية**.

---

## ملفات المشروع
```
nujum-cloud/
├── index.html          ← التطبيق (مربوط بـ Supabase)
├── config.js           ← مفاتيح الاتصال (تعدّلها أنت)
├── supabase-setup.sql  ← سكربت إنشاء قاعدة البيانات
├── vercel.json         ← إعداد النشر
└── .gitignore
```

---

## الخطوة ١: إنشاء قاعدة البيانات في Supabase

1. ادخل إلى **https://supabase.com** واضغط **Start your project** وسجّل دخولك (يمكن بحساب GitHub).
2. اضغط **New project**، اختر اسماً (مثل `nujum`)، وكلمة مرور قوية لقاعدة البيانات، واختر أقرب منطقة جغرافية، ثم **Create new project**. انتظر دقيقة حتى يجهز.
3. من القائمة الجانبية افتح **SQL Editor** ← **New query**.
4. افتح ملف `supabase-setup.sql`، انسخ **كل** محتواه، والصقه في المحرّر، ثم اضغط **Run** (أو Ctrl+Enter). يجب أن تظهر رسالة نجاح.
5. من القائمة الجانبية افتح **Project Settings** (أيقونة الترس) ← **API**. ستجد:
   - **Project URL** — انسخه.
   - تحت **Project API keys**: مفتاح **anon / public** — انسخه أيضاً.

> احتفظ بهاتين القيمتين، ستحتاجهما في الخطوة التالية.

---

## الخطوة ٢: وضع المفاتيح في config.js

افتح ملف `config.js` بأي محرّر نصوص، واستبدل القيمتين:

```js
window.NUJUM_CONFIG = {
  url: "https://xxxxxxxx.supabase.co",   // ضع Project URL هنا
  anonKey: "eyJhbGci..."                 // ضع anon public key هنا
};
```

احفظ الملف. (مفتاح anon عام وآمن للوضع في المتصفّح — هذا طبيعي في Supabase.)

---

## الخطوة ٣: رفع المشروع على GitHub

### الطريقة الأسهل (عبر الموقع — بدون أوامر):
1. ادخل **https://github.com** وسجّل دخولك (أنشئ حساباً مجانياً إن لم يكن لديك).
2. اضغط **+** أعلى اليمين ← **New repository**.
3. اكتب اسماً مثل `nujum`، اجعله **Public**، ثم **Create repository**.
4. في الصفحة التالية اضغط **uploading an existing file**.
5. اسحب كل ملفات مجلّد `nujum-cloud` (index.html, config.js, vercel.json، إلخ) إلى المنطقة المخصّصة.
6. اضغط **Commit changes**.

### أو عبر سطر الأوامر (إن كان Git مثبّتاً):
```bash
cd nujum-cloud
git init
git add .
git commit -m "نُجوم - النسخة السحابية"
git branch -M main
git remote add origin https://github.com/USERNAME/nujum.git
git push -u origin main
```
(استبدل `USERNAME` باسم حسابك.)

---

## الخطوة ٤: النشر على Vercel

1. ادخل **https://vercel.com** واضغط **Sign Up** ← **Continue with GitHub** (سجّل دخولك بحساب GitHub).
2. اضغط **Add New…** ← **Project**.
3. ستظهر مستودعاتك من GitHub. اختر مستودع `nujum` واضغط **Import**.
4. لا تحتاج تغيير أي إعداد (المشروع ملفات ثابتة). اضغط **Deploy**.
5. انتظر دقيقة… ثم ستحصل على رابط مثل `https://nujum.vercel.app` يعمل **من أي مكان وأي جهاز**.

🎉 انتهى! افتح الرابط على جوالك وحاسوبك في الوقت نفسه — أي تغيير من جهاز يظهر فوراً على الآخر.

---

## كيف تجرّب أنه يعمل
- افتح الرابط، ادخل كولي أمر: `parent` / `1234`.
- أضف طفلاً ومهمة.
- افتح الرابط نفسه على جهاز آخر (أو نافذة خفية) وادخل كطفل — ستجد المهمة موجودة.
- عند اعتماد المهمة من جهاز ولي الأمر، تتحدّث شاشة الطفل تلقائياً.

## أسئلة شائعة

**هل المفتاح في config.js يعرّض بياناتي للخطر؟**
مفتاح anon عام بطبيعته. الحماية الحقيقية تأتي من سياسات RLS. الإعداد الحالي مفتوح للاستخدام العائلي البسيط (عائلة واحدة). إن أردت حماية أقوى (عدة عائلات، حسابات منفصلة)، أخبرني لأجهّز لك نسخة بـ Supabase Auth وسياسات RLS مقيّدة.

**أريد تغيير كلمة مرور لوحة الإدارة (1234).**
هي مخزّنة ضمن بيانات البداية. بعد أول تشغيل، عدّلها من قاعدة البيانات (جدول `app_state` ← الحقل `data` ← `parent.pass`)، أو أخبرني لأضيف شاشة لتغييرها.

**كيف أُحدّث الموقع لاحقاً؟**
عدّل الملفات وارفعها على GitHub مجدداً — Vercel ينشر التحديث تلقائياً خلال ثوانٍ.

**ظهر تنبيه «يعمل محلياً فقط».**
معناه أن `config.js` لم تُضبط قيمه بعد، أو أن السكربت في الخطوة ١ لم يُنفّذ بنجاح. تأكّد من نسخ الرابط والمفتاح بشكل صحيح.

---

## ملاحظة على النموذج الحالي
لتبسيط الانطلاق، تُخزَّن حالة العائلة كاملة ككائن JSON واحد في جدول `app_state`، مع مزامنة فورية عبر Realtime. هذا ممتاز لعائلة واحدة وعدد معقول من الأطفال والمهام. للتوسّع إلى عدّة عائلات وآلاف السجلات، يُنصح بالانتقال إلى المخطط العلائقي الكامل (`nujum-schema.sql` من التسليم السابق) مع جداول منفصلة وفهرسة — وأنا جاهز لتجهيز تلك النسخة عند الحاجة.
